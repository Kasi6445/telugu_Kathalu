# Telugu Katalu — Full Project Refactor

You are refactoring an existing Telugu story generation project. The current code is in `generate.py` and related files — please READ them first before making any changes. Do not make assumptions about what exists.

## Project context

- Static website: Telugu moral/folk stories with AI-generated text, images, and TTS audio
- Deployed to Cloudflare Pages via git push (no backend runtime)
- Python 3 script `generate.py` runs locally on Windows 11 to produce assets
- Frontend: pure HTML/CSS/vanilla JS, no framework
- 13 stories exist today (all in folder `stories/` with timestamp IDs like `20260411_171746/`)
- Existing story schema must be preserved for backward compatibility

## Current problems to fix

1. **Category balancing bug:** All 13 stories ended up in `neeti` category. The "pick least populated" logic falls through to the first dict key on ties. Fix with `random.choice()` among tied candidates.
2. **Story quality is mid:** Using Groq + llama-3.3-70b. Telugu output lacks cultural depth. JSON sometimes breaks.
3. **Image gen broken:** Leonardo AI credits exhausted.
4. **TTS is paid:** Using Sarvam AI. Want to switch to Google Cloud TTS free tier.
5. **No subcategories:** Topics inside categories are flat. Need hierarchical structure.
6. **Search stub:** Shows "Coming soon" alert.
7. **No share / OG tags / structured data:** Poor social sharing and SEO.

## Target architecture

### Stack migration

- **Story generation:** Groq → Gemini 2.5 Flash via `google-generativeai` library. Use `response_mime_type="application/json"` to force valid JSON output. Read key from `GEMINI_API_KEY` env var.
- **Image generation:** Leonardo → Imagen 3 via Vertex AI (package: `google-cloud-aiplatform`). Use service account at `GOOGLE_APPLICATION_CREDENTIALS`. Target size 1024×768 landscape. If Imagen blocks a prompt on content policy, retry with a softer prompt auto-generated by Gemini.
- **TTS:** Sarvam → Google Cloud TTS Chirp3 HD voices in Telugu (`te-IN`). Use `google-cloud-texttospeech` library. Per-category voice map (see below). Fallback to `te-IN-Standard-B` with `pitch=-2.0, speaking_rate=0.9` if Chirp3 fails.

### Phase 2: New generation pipeline
